# Bulk RNA-seq Pipeline

A bulk RNA-seq analysis pipeline for paired-end data, from raw FASTQ
files to differential expression results. Designed to run on an HPC
cluster with a SLURM scheduler and Singularity containers.

## Pipeline overview

```
Raw FASTQ
   │
   ▼
01_trim_galore.sh        → adapter trimming and quality filtering
   │
   ▼
02_STAR_alignment.sh     → alignment to the reference genome (2-pass)
   │
   ▼
03_featurecounts.sh      → gene-level read quantification
   │
   ▼
04_DEA_DESeq2.R           → differential expression, PCA, volcano, heatmaps
```

## Requirements

- Access to a cluster with SLURM and Singularity/Apptainer
- Singularity images (`.sif`) for:
  - [Trim Galore](https://github.com/FelixKrueger/TrimGalore) (used: v0.6.6)
  - [STAR](https://github.com/alexdobin/STAR) (used: v2.7.8a)
  - [samtools](http://www.htslib.org/) (used: v1.15)
  - [Subread/featureCounts](https://subread.sourceforge.net/) (used: v2.0.1)
- A pre-built STAR index for your reference genome
- A GTF annotation file (the same version used to build the STAR index)
- R (≥4.2) with the packages: `DESeq2`, `ashr`, `ggplot2`, `ggrepel`,
  `pheatmap`, `AnnotationDbi`, and the annotation package for your
  organism (e.g. `org.Mm.eg.db`, `org.Hs.eg.db`)

## Usage

Each script has a configuration section at the top marked with
`# CHANGE:` comments — these are the only lines you need to edit to
adapt the pipeline to a new project. See
[`docs/CONFIGURATION.md`](docs/CONFIGURATION.md) for details on each
variable.

### 1. Trimming

```bash
sbatch scripts/01_trim_galore.sh
```

Requires an `input_files.txt` inside the raw FASTQ folder, listing
R1/R2 filenames alternately (one sample every two lines). See
`docs/CONFIGURATION.md` for the exact format.

### 2. Alignment

```bash
bash scripts/02_STAR_alignment.sh
```

Automatically detects samples from the trimmed FASTQ files
(convention `<sample>_1_val_1.fq.gz` / `<sample>_2_val_2.fq.gz`,
generated by Trim Galore) and submits one SLURM job per sample.

> **Note:** if your sample names contain `+` or `-` (e.g. condition
> names like `treatment+` / `treatment-`), double-check that these
> characters are preserved correctly in the resulting BAM names —
> some downstream steps (reading into R) may convert them to `.`
> when parsing column names. See `docs/TROUBLESHOOTING.md`.

### 3. Quantification

```bash
sbatch scripts/03_featurecounts.sh
```

Generates a single quantification matrix for all samples
(`<PROJECT>_gene_quantification.txt`), with the first 6 columns
being featureCounts metadata and the rest one column per BAM.

### 4. Differential expression

```r
# From RStudio, after editing the configuration section of the script
source("scripts/04_DEA_DESeq2.R")
```

For each comparison defined in `COMPARISONS`, generates in its own
folder:

| File | Description |
|---|---|
| `DESeq2_results_all.csv` | Full DESeq2 results (all genes) |
| `GSEA_ranked.rnk` | Genes ranked by log2FC, no header, for GSEA preranked |
| `Volcano_unannotated.png` | Volcano plot, unlabeled |
| `Volcano_top10annotated.png` | Volcano plot with top significant genes labeled |
| `Heatmap_sig_genes.png` | Heatmap of significant genes (up + down) |
| `genes_upregulated.csv` / `genes_downregulated.csv` | Separate lists of DEGs |

In addition, at the global level: `PCA/PCA_global.png`,
`normalized_counts.csv` and `VST_matrix.csv` (both with gene symbol
instead of Ensembl ID).

## Design notes

- Scripts 01-03 are SLURM/Singularity-first: they are meant for HPC
  clusters, not for direct local execution.
- `02_STAR_alignment.sh` and `03_featurecounts.sh` expect absolute
  paths accessible from both the login node and the compute nodes
  (Singularity bind path).
- `04_DEA_DESeq2.R` is species- and design-agnostic: just fill in
  `sample_columns`, `COMPARISONS`, and the matching annotation
  package.
- If your experimental design includes paired replicates (same
  animal, passage, or donor across conditions), add a replicate
  factor to the DESeq2 design (`~ replicate + condition`) to control
  for that variability — see comments in the script itself.
